# Decisions

ADR (Architecture Decision Record) untuk keputusan teknis yang berdampak panjang. Setiap ADR menjelaskan: konteks, opsi yang dipertimbangkan, keputusan, dan konsekuensi.

**Naming:** `ADR-NNNN-<kebab-slug>.md` (NNNN sequential, mulai 0001).

**Status flow:** `proposed` → `accepted` → (`superseded` | `deprecated`).

> Folder ini masih kosong. ADR pertama dibuat saat ada keputusan arsitektural yang perlu dirujuk lintas-session.

## Kapan Dibaca

- Onboarding ke project / workspace
- Sebelum re-open diskusi yang pernah diputuskan
- Saat melakukan refactor besar yang mungkin melanggar keputusan lama
